AGY CLI & Slash Commands

สรุปคำสั่งลัดและคำสั่งใช้งานทั้งหมดใน Antigravity CLI (agy) พร้อมระบบจำลองสถานการณ์ตัวอย่าง (Scenario Simulator) เพื่อทำความเข้าใจการใช้งานจริง

1. คำสั่งหน้า Terminal (AGY CLI Subcommands & Flags)
agy เริ่มระบบ TUI

เปิดใช้งาน Antigravity CLI ในรูปแบบ Interactive Terminal UI (TUI) สำหรับพิมพ์โต้ตอบ สั่งงาน AI Pair Programmer

agy run "[prompt]" รันงานทันทีแบบ Non-Interactive

สั่งรันงานสั้นๆ จาก Terminal Command Line โดยไม่ต้องเปิดเข้าโหมด TUI เหมาะสำหรับการสคริปต์อัตโนมัติ

จำลองสถานการณ์: agy run
📌 สถานการณ์
คุณต้องการสั่งให้ AI ตรวจเช็กไฟล์ README.md หรือแก้บั๊กสั้นๆ จาก Bash script / CI Pipeline โดยไม่ต้องเข้าหน้า TUI
💬 คำสั่งที่ใช้
agy run "ช่วยตรวจสอบไฟล์ index.html แล้วแก้คำผิดให้เรียบร้อย"
🤖 พฤติกรรมของระบบ
AGY จะทำงานในเบื้องหลัง แก้ไขไฟล์ แล้วพิมพ์สรุปผลออกมาหน้า Terminal ทันทีแล้วจบการทำงาน
agy task [list | status | kill] จัดการ Background Tasks

ตรวจสอบและจัดการงานที่รันอยู่ในฉากหลัง (Background Process) เช่น คำสั่งเทสต์ หรือเซิร์ฟเวอร์ที่รันค้างไว้

จำลองสถานการณ์: agy task
📌 สถานการณ์
คุณสั่งให้รันเซิร์ฟเวอร์รันค้างไว้ในฉากหลัง แล้วต้องการเช็กดูว่า Task นั้นยังทำงานอยู่ไหม หรือสั่งปิด Task นั้น
💬 คำสั่งที่ใช้
agy task list
agy task kill task-123
agy agent [list | kill] จัดการ Subagent ลับ

ตรวจสอบรายการ Subagent ทั้งหมดที่ถูกเรียกใช้งานในฉากหลัง และสั่งยุติการทำงานของ Subagent ตาม ID

2. คำสั่ง Slash Commands ในหน้าแชท (TUI / Slash Commands)
/goal รันงานยาวข้ามคืนแบบละเอียดยิบ

โหมดลุยงานระยะยาว ให้ AI ทำงานต่อเนื่องไม่หยุดจนกว่าเป้าหมายใหญ่จะสำเร็จ 100% เหมาะสำหรับงาน Refactor ใหญ่

จำลองสถานการณ์: /goal
📌 สถานการณ์
คุณต้องการสั่งให้ AI เขียน Unit Test ครอบคลุมทั้งโปรเจกต์ 50 ไฟล์ และปล่อยให้รันแก้ไขไปเรื่อยๆ ข้ามคืน
💬 คำสั่งที่ใช้
/goal เขียน Unit Test ให้ครบทุกโมดูลในโฟลเดอร์ tests/ และห้ามหยุดจนกว่าทุกไฟล์จะผ่าน 100%
🤖 พฤติกรรมของระบบ
AI จะวนลูปแก้โค้ด รันเทสต์ ปรับปรุงข้อผิดพลาดอัตโนมัติ โดยไม่หยุดถามเรื่องเล็กน้อย จนกว่าเป้าหมายจะสำเร็จลุล่วง
/schedule ตั้งเวลา / Cron Job อัตโนมัติ

ตั้งเวลานาฬิกาปลุก หรือตั้ง Cron Schedule ให้ AI กลับมาเช็กงานตามรอบเวลา (เช่น ทุกๆ 5 นาที)

จำลองสถานการณ์: /schedule
📌 สถานการณ์
คุณเพิ่งสั่ง Deploy หน้าเว็บขึ้น GitHub Pages แล้วอยากให้ AI กลับมาตรวจเช็ก URL ว่าออนไลน์หรือยังในอีก 10 นาที
💬 คำสั่งที่ใช้
/schedule ในอีก 10 นาที ให้ลองยิง HTTP request ไปที่หน้าเว็บว่าขึ้น 200 OK หรือยัง
/plan วางแผนสเต็ปการทำงานก่อนแตะโค้ด

บังคับให้ AI ทำการวิเคราะห์ระบบ วางแผนเป็นข้อๆ (Blueprint Plan) ให้คุณอนุมัติก่อนที่จะลงมือแก้ไขไฟล์จริง

/grill-me สัมภาษณ์ปรับความเข้าใจกับผู้ใช้

ให้ AI ยิงคำถามสัมภาษณ์คุณยิบๆ (Interview Mode) เพื่อเก็บความต้องการและตัดสินใจเรื่อง Architecture ให้ชัดเจนก่อนเริ่มงาน

จำลองสถานการณ์: /grill-me
📌 สถานการณ์
โจทย์โปรเจกต์ยังกว้างๆ เช่น "อยากได้ระบบล็อกอิน" AI จะยิงคำถามถามคุณทีละข้อ (ใช้ OAuth หรือ Password?, มี Roles ไหม?) จนกว่าข้อสรุปจะเป๊ะ
/teamwork-preview กระจายงานให้ Subagents ทีมใหญ่

แตกงานออกเป็นหลายๆ ทีม แล้วสั่งให้ Subagents ทำงานขนานกันไปพร้อมๆ กัน (Frontend, Backend, Tester)

/learn บันทึกความจำ / กฎระเบียบใหม่

บันทึกคำติชมหรือสไตล์การเขียนโค้ดของคุณลงในระบบความจำ (`.gemini/rules/`) เพื่อให้ AI จำไว้ใช้ตลอดไป

/help | /clear | /exit คำสั่งควบคุมพื้นฐาน

/help: ดูคำสั่งที่มีทั้งหมด | /clear: ล้างประวัติแชทเริ่มต้นใหม่ | /exit: ออกจากโปรแกรม
